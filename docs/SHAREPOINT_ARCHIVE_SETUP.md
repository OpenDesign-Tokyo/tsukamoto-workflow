# SharePoint 自動アーカイブ セットアップ手順

> **目的**: 承認完了した申請の PDF を SharePoint に自動保存し、申請者・承認者などのメタデータを SharePoint プロパティに保持してソート・絞り込み可能にする。
>
> **対象環境**: Microsoft 365 / Entra ID + SharePoint Online

---

## 1. Azure AD アプリへの権限付与

既存の Entra ID アプリ（テナント `97641f9f-2d1e-4f7c-a82d-7b2edad4232c`、クライアント `9e5b2fa9-765b-4aea-b691-f521da7d377d`）に以下を追加。

1. [Azure ポータル](https://portal.azure.com/) → **Microsoft Entra ID** → **アプリの登録** → 該当アプリを選択
2. 左メニュー **API のアクセス許可** → **アクセス許可の追加**
3. **Microsoft Graph** → **アプリケーションの許可**（委任ではなく）
4. 以下にチェックを入れて追加:
   - `Sites.ReadWrite.All` — SharePoint サイト内ファイルの読み書き
5. 一覧に追加後、上部の **<テナント名> に管理者の同意を与えます** ボタンをクリック

> 既存の `User.Read.All`（組織同期用）はそのまま残しておくこと。

---

## 2. SharePoint サイト・ライブラリのカラム設定

SharePoint 側でメタデータをソート可能にするため、ドキュメントライブラリにカスタム列を追加します。

### 対象ライブラリ
ユーザー作成の SharePoint フォルダ（例: `tsukamoto365.sharepoint.com/sites/msteams_42ae0e` の「申請ワークフロー」配下）

### 追加するカラム

ライブラリの **+ 列を追加** から、以下の **内部名（半角英数字なし → 日本語表示名のまま登録可）** で作成:

| 列名（表示名 = 内部名） | 種類 | 必須 | 用途 |
|---|---|---|---|
| 申請番号 | 1 行テキスト | ✓ | 例: `APP-2026-0001` |
| 申請者 | 1 行テキスト | ✓ | 申請者氏名 |
| 最終承認者 | 1 行テキスト |  | 最終承認ステップの承認者 |
| 承認者一覧 | 複数行テキスト |  | 全承認者を `ステップ: 名前 / ステップ: 名前` で連結 |
| 申請日 | 日付と時刻 |  | 申請提出日 |
| 承認完了日 | 日付と時刻 | ✓ | 最終承認完了日 |
| 書類種別 | 1 行テキスト |  | 例: `企画外注向け注文書` |

> **注意**: 列を追加する前に PDF アップロードを行ってもファイルは保存されます（メタデータ設定だけスキップされる）。列追加後に再アップロードすればメタデータが反映されます。

### フォルダ構成

各書類種別と同じ名前のフォルダをライブラリ直下に作成してください（ユーザー側で実施済み）:
- `企画外注向け注文書`
- `採寸申請書`
- `有給休暇-半休申請書`
- （以降、新規書類種別を追加する都度フォルダも追加）

> フォルダ名は `document_types.name` カラムと**完全一致**が必要です。

## 3. Vercel 環境変数の設定

Vercel ダッシュボード → プロジェクト → **Settings** → **Environment Variables** に以下を追加:

| 変数名 | 値の例 | 説明 |
|---|---|---|
| `MS_GRAPH_CLIENT_ID` | `9e5b2fa9-765b-4aea-b691-f521da7d377d` | Azure AD クライアント ID（既存と同じ） |
| `MS_GRAPH_CLIENT_SECRET` | `(シークレット)` | Azure AD クライアントシークレット |
| `AZURE_AD_TENANT_ID` | `97641f9f-2d1e-4f7c-a82d-7b2edad4232c` | テナント ID（既存と同じ） |
| `SHAREPOINT_HOSTNAME` | `tsukamoto365.sharepoint.com` | SharePoint のホスト名 |
| `SHAREPOINT_SITE_PATH` | `msteams_42ae0e` | サイトのパス（`/sites/` の後ろの部分） |
| `SHAREPOINT_ROOT_FOLDER` | `申請ワークフロー` | ライブラリ直下のフォルダ名（任意・省略時はライブラリ直下） |

> **値の確認方法**:
> - `SHAREPOINT_HOSTNAME` / `SHAREPOINT_SITE_PATH`: SharePoint で対象サイトを開き、URL `https://tsukamoto365.sharepoint.com/sites/msteams_42ae0e` のホスト部分とサイトパス部分
> - クライアントシークレットは Azure ポータル → アプリ → **証明書とシークレット** から生成（有効期限を 24 か月推奨）

設定後、Vercel で **再デプロイ** を実行して環境変数を反映してください。

---

## 4. 動作確認

1. テスト用に申請を1つ最終承認まで進める
2. 数秒〜数十秒後に SharePoint の該当フォルダ（例: `企画外注向け注文書`）を確認
3. `APP-XXXX-NNNN_<タイトル>.pdf` が保存されていること
4. ファイルのプロパティペインに上記カラムの値が入っていること
5. アプリケーション詳細画面で `sharepoint_url` が記録され、SharePoint へのリンクが表示されること
6. 一覧ビューで列ヘッダーをクリックしてソートできること

---

## 5. トラブルシューティング

| 症状 | 原因 / 対処 |
|---|---|
| ファイルが SharePoint に出ない | Vercel ログで `[archive]` ログを確認。`SharePoint 未設定のためスキップ` が出ていれば環境変数の見直し |
| `フォルダ "XXX" が見つかりません` エラー | 書類種別名と SharePoint フォルダ名の完全一致を確認（全角/半角・スペース・記号） |
| メタデータが反映されない（ファイルは保存される） | ライブラリにまだカラムを追加していない。手順 2 の列を追加後、新規申請で再確認 |
| `401` / `Forbidden` | Azure AD アプリの `Sites.ReadWrite.All` 権限と管理者同意の付与を確認 |
| トークン取得失敗 | クライアントシークレットの有効期限切れ可能性。Azure ポータルで再発行 |

---

## 6. セキュリティ・運用メモ

- アーカイブは **失敗してもワークフロー本体には影響しない** 設計（fire-and-forget）。失敗時は `audit_logs` に `phase: archive.*` で記録されるため、ダッシュボードから追跡可能。
- 監査要件で「PDF が必ず SharePoint に保存される」ことが必要な場合は、cron で再試行を組む拡張を Phase 2 で追加する。
- クライアントシークレットは 90 日 〜 24 か月でローテーション。Azure ポータルから新シークレット発行 → Vercel 環境変数更新 → 旧シークレット失効、の順で実施。
